# Security Policy

## Supported Versions

The following versions of PowerShell-Automation-Toolkit are currently supported with security updates.

| Version | Supported |
|----------|-----------|
| 1.x | ✅ |
| < 1.0 | 🚧 Development |

---

## Reporting a Security Issue

If you discover a security vulnerability, please do **not** open a public GitHub issue.

Instead:

1. Contact the project maintainer.
2. Provide a detailed description of the issue.
3. Include steps to reproduce the problem, if applicable.
4. Allow reasonable time for the issue to be investigated before public disclosure.

---

## Response Process

Security reports will be:

- Acknowledged as soon as practical
- Investigated promptly
- Fixed in a future release where appropriate

---

## Scope

This policy applies to:

- PowerShell modules
- Shared helper functions
- GitHub workflows
- Documentation examples
- Configuration files

---

Thank you for helping improve the security of PowerShell-Automation-Toolkit.
